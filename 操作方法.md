# 操作方法

目的は完成表を作ることなので、通常は番号を選びません。

```bash
cd /Volumes/SSD_External/Business/Materials/2026-06_有報自動抽出/yuho_auto_extract
./yuho
```

これだけで、EDINET DB作成、抽出、正規化、検証、レビューキュー分割、完成表、分析表、実行レポートまで更新します。

出力先:

```text
data/review/review_queue.xlsx
data/review/review_resolved_local_pass.xlsx
data/review/review_queue_local_needs_manual.xlsx
data/final/final_master_long.csv
data/final/final_master_wide.xlsx
data/final/analysis_dataset.xlsx
data/final/run_report.md
```

必要なAPIキーはEDINETだけです。既に有報をダウンロード済みなら、完成表更新ではAPIキーを使いません。

```text
EDINET_API_KEY=...
```

補助操作が必要な時だけメニューを開きます。

```bash
./yuho menu
```

短縮コマンド:

```bash
./yuho run
./yuho test
./yuho init
./yuho report
./yuho audit
./yuho web
./yuho cli --help
```

ブラウザで操作したい場合は、次を実行して `http://127.0.0.1:8765` を開きます。

```bash
./yuho web
```

一度保存したレビューを修正したい場合は、レビュー画面で `保存済みのみ` に絞り込み、対象行をクリックします。保存済みの判定・修正値・メモが右側の編集欄に入るので、直して `上書き保存` を押します。

保存済みレビューを削除したい場合も同じく対象行を選び、`レビュー削除` を押します。削除されるのは保存済み判断だけで、レビュー候補行は残ります。

レビュー画面では、修正値とあわせてメモに `LOC`, `TABLE`, `LABEL`, `XBRL_TAG`, `RULE_HINT` を残します。`レビューから候補作成` を押すと、同じ `field_id` のメモを集計して次のファイルを作ります。

```text
data/review/rule_candidates.csv
```

候補作成だけでは抽出設定を直接書き換えません。候補行をクリックして内容を確認し、`選択候補を設定へ反映` を押すと、その `field_id` について次を追記します。

- `config/field_definition.csv` / `config/field_definition.xlsx`: XBRLタグ候補、セクションキーワード、行ラベル候補
- `config/extraction_sections.yml`: `review_<field_id>` のローカル表探索セクション

既存値は消さずに追記し、反映前には `.bak-YYYYMMDDTHHMMSSZ` のバックアップを作ります。反映後にWebの実行画面で `レビュー学習後に再取得` を押すと、同じ項目を他年度・他社でも再取得し、その後に `review_resolved.csv` の保存済みレビューも最終出力へ反映します。

`保存値を最終結果に反映` は、既に保存済みのレビュー値を `final_master_wide.csv` などへ入れ直すための処理です。抽出ルールの更新や再取得は行いません。反映に成功すると `review_resolved.csv` の該当行に `applied_status`, `applied_value`, `applied_at` が記録され、Webのレビュー一覧にも反映状態が表示されます。レビューは根拠・再出力・再取得に使う監査ログなので、反映後も自動削除しません。取り消す場合だけ `レビュー削除` を使います。

ルールを育てて取り直す場合は、`レビューから候補作成` → `選択候補を設定へ反映` → `レビュー学習後に再取得` の順で実行します。

レビュー由来ルールが増えてきたら、WebのAI分析画面または実行画面で `アルゴリズム監査パック生成` を押します。CLIでは次でも生成できます。

```bash
./yuho audit
./yuho cli build-algorithm-audit
```

生成先は `data/algorithm_audit/` です。`ALGORITHM_AUDIT_PROMPT.md` をAIに渡すと、`review_*` セクションの増えすぎ、証跡1件だけの全社共通化、低カバレッジ項目、XBRL候補や同義語の広がりすぎを監査できます。AIの提案はそのまま反映せず、実装前に人間が確認します。

手確認が必要な候補は、次のファイルを見ます。

```text
data/review/review_queue_local_needs_manual.xlsx
```

レビュー済みファイルを手で反映する場合:

```bash
python -m yuho_auto_extract export-final --reviewed data/review/review_resolved.xlsx
python -m yuho_auto_extract build-analysis
python -m yuho_auto_extract report
```
